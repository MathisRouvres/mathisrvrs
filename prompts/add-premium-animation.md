## Objectif

Rendre l'expérience plus fluide, moderne et premium sans nuire à la lisibilité, la performance ou l'accessibilité.

## Contraintes

- Ne change pas la logique métier.
- Ne rajoute pas de dépendance si le projet a déjà CSS, Framer Motion, GSAP ou une solution existante.
- Utilise l'outil d'animation le plus simple possible.
- Ajoute un fallback prefers-reduced-motion.
- Évite les animations longues ou trop visibles.
- Préserve le responsive.

## Attendu

1. Inspecte les composants/styles existants.
2. Propose le type de motion approprié.
3. Implémente proprement.
4. Donne les étapes de test.
5. Indique les risques de performance ou d'accessibilité.
